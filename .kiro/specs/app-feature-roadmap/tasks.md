# Implementation Plan: App Feature Roadmap

## Overview

Implementation follows the priority order established in requirements: onboarding flow first, then alert banner, image upload, secure share links, safe migrations, service worker, and finally the strategic vector/suggestion features. All DB changes go through `db/schema.sql` + `db/queries.sql` → `sqlc generate`. No files in `internal/db/` are ever edited manually.

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- Strategic features (tasks 17–19) are fully optional and deferred
- Each task references specific requirements for traceability
- Property tests use `pgregory.net/rapid` with a minimum of 100 iterations each
- DB changes always follow: edit `db/schema.sql` + `db/queries.sql` → `sqlc generate` — never edit `internal/db/` directly
- Goose migrations in `db/migrations/` are numbered sequentially from `00009`
